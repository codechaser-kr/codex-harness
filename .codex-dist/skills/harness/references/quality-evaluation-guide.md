# Quality Evaluation Guide (Codex용 하네스 품질 비교 기준)

이 문서는 실행 하네스가 "구조가 존재하는가"를 넘어,  
하네스가 없는 상태보다 어떤 품질 차이를 만드는지 비교하는 기준을 정의한다.

---

## 핵심 전제

- 구조 검증만 통과했다고 좋은 하네스는 아니다.
- 좋은 하네스는 시작 역할 판단, 질문 수, handoff 명확성, 재진입 기준, 검증 가능성에서 차이를 만든다.
- 품질 평가는 가능하면 `without-skill` 기준선과 `with-skill` 결과를 나란히 놓고 본다.

---

## 1. 비교 대상

품질 비교는 최소 두 상태를 둔다.

- `without-skill`: 하네스가 없거나, 하네스 없이 일반적인 판단만 하는 상태
- `with-skill`: 현재 하네스 구조, 탐색 문서, 역할 정의, 스킬 정의를 사용한 상태

핵심은 "문서가 더 많아졌는가"가 아니라  
"판단 품질이 실제로 달라졌는가"를 보는 것이다.

---

## 2. 최소 비교 항목

다음 다섯 가지는 항상 비교한다.

### 1) 시작 역할 판단

- without-skill: 시작 역할이 모호하거나 한 번에 여러 역할을 제안하는가
- with-skill: 현재 시작 역할 1개가 더 선명해졌는가

### 2) 사용자 질문 절제

- without-skill: 질문이 과도하거나 추상적인가
- with-skill: 추가 질문이 0~2개 수준으로 줄고, 실제 탐색 부족 지점에 맞는가

### 3) handoff 명확성

- without-skill: 다음 역할이나 재진입 경로가 흐릿한가
- with-skill: 다음 역할, 보강 역할, 재진입 Phase가 더 분명한가

### 4) 저장소 근거 연결

- without-skill: 일반론이나 역할 설명만 반복하는가
- with-skill: 루트 기준 저장소 재독해와 사용자 입력이 실제 판단과 연결되는가

### 5) 검증 가능성

- without-skill: 결과가 애매해서 검증하기 어려운가
- with-skill: verify나 validator가 어느 지점을 점검해야 하는지 더 명확한가

---

## 3. 선택 비교 항목

프로젝트에 따라 다음도 본다.

- 운영 모드 판단이 더 빨라졌는가
- 재구성 필요 여부를 더 빨리 드러내는가
- AGENTS.md 같은 상위 컨텍스트 충돌을 더 빨리 드러내는가
- 반복 흐름이 템플릿 후보로 더 잘 축적되는가

---

## 4. 평가 결과 표현

평가는 긴 감상이 아니라 짧은 비교 문장으로 남긴다.

예:

- with-skill에서는 시작 역할이 `domain-analyst`로 고정되고, without-skill보다 초기 질문 수가 줄었다.
- with-skill에서는 AGENTS.md 충돌을 update 전에 드러내지만, without-skill에서는 뒤늦게 문서 불일치로 드러난다.

즉 "좋아졌다"보다  
"무엇이 어떻게 달라졌는가"를 남긴다.

---

## 5. 적용 위치

이 기준은 다음 위치에서 사용한다.

- `run-harness` 출력 계약 점검
- `validator` 피드백 문구 점검
- 회귀 테스트에서 최소 품질 비교 기준 점검
- 메타시스템 성숙도 회고 시 `meta-system-maturity-guide.md`와 함께 비교 기준으로 사용

---

## 6. 다른 레퍼런스와의 연결

- `agents-sync-guide.md`: 상위 컨텍스트 정렬이 품질 비교에서 어떤 차이를 만드는지 연결한다.
- `phase-selection-matrix.md`: 재진입 기준이 with-skill 상태에서 더 선명해지는지 비교할 때 사용한다.
- `skill-testing-guide.md`: 구조 검증 외에 어떤 품질 비교를 자동화할지 연결한다.
- `meta-system-maturity-guide.md`: 구조 존재를 넘어 운영 가능한 메타시스템 수준인지 판정할 때 연결한다.
